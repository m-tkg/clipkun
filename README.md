# clipkun

Mac で動くクリップボード履歴ツール。メニューバーに常駐し、ホットキーでカーソル位置に
クリップボード履歴をポップアップ表示して、過去のコピー内容をすばやく呼び出せます。

テキスト・ファイル・画像など多様な種類を履歴に残し、画像はサムネイルで一覧表示します。

## 特長

- **ホットキーでカーソル位置にポップアップ**（既定 <kbd>⌥V</kbd>、設定で変更可）
- 各行に**番号付き一覧**（最新＝1）。<kbd>↑</kbd>/<kbd>↓</kbd> やクリックで選択、<kbd>Enter</kbd> で確定、<kbd>Esc</kbd>／外側クリックでキャンセル
- 選んだ履歴を最新のクリップボードへ書き戻し。**テキストはそのまま前面アプリへ自動ペースト**
- **テキスト・ファイル・画像**に対応。画像はサムネ表示
- **保持期間**を設定可能（既定1日 / 1〜23時間・1〜30日）。期限切れは自動削除
- **最大件数**の上限（既定200件）。超過分は古いものから削除
- メニューバーから**履歴の全削除**、各行のゴミ箱で**個別削除**
- 同じ内容を再コピーすると重複させず先頭へ移動
- ログイン時の自動起動、GitHub Releases からの自己アップデートに対応

## 動作環境

- macOS 13 以降

## インストール

[Releases](https://github.com/m-tkg/clipkun/releases) から最新の `Clipkun.zip` をダウンロードして展開し、
`Clipkun.app` を `/Applications` に移動して起動してください（署名＋公証済み）。

起動するとメニューバーにクリップボードのアイコンが表示されます（Dock には表示されません）。

## 使い方

1. 普段どおりテキスト・ファイル・画像をコピーすると履歴に貯まります
2. <kbd>⌥V</kbd> を押すとカーソル位置に履歴ポップアップが出ます
3. <kbd>↑</kbd>/<kbd>↓</kbd> またはクリックで選び、<kbd>Enter</kbd> で確定
   - テキストはその場のアプリに自動で貼り付けられます
   - ファイル・画像はクリップボードに入るので <kbd>⌘V</kbd> で貼り付けます
4. メニューバーアイコン →「設定…」でホットキー・保持期間・最大件数を変更できます

## 権限について

clipkun は機能に応じて以下の権限を求めることがあります（初回利用時にダイアログが出ます）。

- **アクセシビリティ** … テキストの自動ペースト（⌘V の合成送出）に必要
- **ファイルとフォルダ** … 保護フォルダ（ダウンロード / デスクトップ / 書類 等）にあるファイルを
  履歴から貼り付ける場合に必要

権限を与えなくてもクリップボードへの書き戻し自体は動作します（その場合は手動で <kbd>⌘V</kbd>）。

## 開発

Swift Package Manager の2ターゲット構成です。

- `ClipkunCore` … AppKit/Carbon に依存しない純粋ロジック（履歴モデル・保持期間・配置計算・設定）。TDD 対象
- `Clipkun` … メニューバー常駐・クリップボード監視・ポップアップ・設定 UI などの App 本体

```sh
# テスト（純粋ロジック）
swift test

# ローカル検証用の .app をビルド（Developer ID 署名、本番と別バンドルID）
LOCAL=1 bash Scripts/bundle.sh debug
open "Clipkun (Local).app"
```

ローカルビルドはバンドルID `com.mtkg.clipkun.local`・表示名「Clipkun (Local)」で本番と分離され、
メニューバーに「ローカル」と併記されます。

### リリース

`Resources/Info.plist` の `CFBundleShortVersionString` を上げて `main` に push すると、GitHub Actions が
テスト → Developer ID 署名 → 公証 → `v<version>` リリース作成までを自動で行います（同名タグがあればスキップ）。
